# Transformer

This repo contains two takes on the Transformer architecture. `attention.py` implements a single Transformer block from scratch in NumPy — multi-head self-attention, a two-layer feed-forward network with ReLU, and layer normalization wired together with residual connections. `train.py` is the same idea rebuilt in PyTorch as a `TinyTransformer` `nn.Module`: an embedding layer feeds into `nn.MultiheadAttention`, then an FFN, with `LayerNorm` and residuals around each, and a final linear projection back to vocabulary logits over 27 tokens.

Building this clarified what each piece actually does underneath the framework abstractions. Single-head attention is a similarity score between every pair of tokens (`QKᵀ`), normalized by softmax and used to mix value vectors — scaling by `√d_k` keeps softmax from saturating. Multi-head attention runs several of these in parallel so each head can specialize on a different kind of relationship; concatenating and projecting back mixes those perspectives. The feed-forward network transforms each token independently through a wider hidden space, layer norm keeps the residual stream numerically stable, and residual connections let gradients flow through deep stacks. Porting the NumPy version to PyTorch also made the shape conventions concrete — adding a sequence dimension with `unsqueeze`, threading it through attention, and projecting back to logits.

`attention.py` exports the NumPy functions (`softmax`, `attention`, `multihead_attention`, `transformer_block`) without running any code at import time — the demo block was removed so the file can be imported as a module.

## What `train.py` does

**The model.** `TinyTransformer` is a one-block transformer over a character vocabulary. Characters are mapped to integer IDs and embedded into 8-dimensional vectors. Learned positional embeddings (up to sequence length 10) are added before the attention block so the model can distinguish token order. A 4-head self-attention layer lets each token attend to the others, followed by a 2-layer feed-forward network (8 → 32 → 8) with ReLU. Both sublayers are wrapped in residual connections and `LayerNorm`. A final linear layer projects each token's 8-dim representation to a 27-way logit vector (one score per vocabulary slot).

**The training loop.** The string `"hello world"` is tokenized; the model is trained to predict each character from the previous one — `x_train` is every character except the last, `y_train` is every character except the first. For 500 epochs we run a forward pass, compute cross-entropy loss between the predicted logits and the target IDs, backpropagate, and step Adam (lr=0.01). Loss is printed every 50 epochs so you can watch it fall toward zero as the model memorizes the sequence.

**The output.** After training, the script runs the model in eval mode on the same input and prints two lines: the input characters and the model's predicted next character at each position. On a successful run the predicted string matches `text[1:]` (`"ello world"`), confirming the model has learned the character-level transitions in its tiny training set. This is overfitting by design — the goal is to verify the architecture trains end-to-end, not to generalize.
