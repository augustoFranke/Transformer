# Transformer

This code implements a single Transformer block from scratch in NumPy with multi-head self-attention, a two-layer feed-forward network with a ReLU nonlinearity, and layer normalization wired together with residual connections. Multiple attention heads run in parallel, their outputs are concatenated and projected back through `W_O`, and the block computes `LayerNorm(X + MultiHeadAttention(X))` followed by `LayerNorm(X + FeedForward(X))` — the core unit stacked repeatedly in real Transformer models.

Building this clarified what each piece actually does underneath the framework abstractions. Single-head attention is a similarity score between every pair of tokens (`QKᵀ`), normalized by softmax and used to mix value vectors — scaling by `√d_k` keeps softmax from saturating. Multi-head attention runs several of these in parallel with independent `W_Q`, `W_K`, `W_V` per head, so each head can specialize on a different kind of relationship; concatenating and projecting with `W_O` mixes those perspectives back into a single representation. The feed-forward network then transforms each token independently through a wider hidden space, layer norm keeps the residual stream numerically stable, and the residual connections let gradients flow through deep stacks.

The output is a matrix where each row is an updated representation of the corresponding input token — enriched with context from the rest of the sequence via multi-head attention and then transformed nonlinearly by the feed-forward layer. Running the script prints the result of passing a random 4-token, 8-dimensional input through one such block with 4 attention heads.
